# 3. Hybrid Search — Combining Vector + Keyword Search

---

## ELI5

Imagine you're searching for a recipe.

**Vector search** understands meaning: "how to make Italian pasta" finds "spaghetti carbonara recipe" even though the words don't match.

**Keyword search** matches exact words: searching "carbonara" finds all documents containing that exact word.

Neither is perfect alone:
- Vector search might miss documents with rare technical terms
- Keyword search misses synonyms and semantic matches

**Hybrid search** uses BOTH — it finds documents that are semantically similar AND contain the right keywords. This is why your CLAUDE.md says "MANDATORY IN PROD".

---

## Why Hybrid Search Matters

### Where Vector Search Fails

```
Query: "error code E-4021"

Vector search thinks: "something about errors and codes"
  → Returns: general troubleshooting guides, error handling best practices
  → Misses: the specific document about E-4021

Keyword search: finds "E-4021" exactly
  → Returns: the exact document you need
```

### Where Keyword Search Fails

```
Query: "how to make my Python code faster"

Keyword search: looks for "Python", "code", "faster"
  → Returns: documents containing those exact words
  → Misses: articles about "performance optimization" or "profiling"

Vector search: understands the meaning
  → Returns: articles about profiling, caching, asyncio, etc.
```

### Hybrid Combines Both Strengths

```
Query: "error code E-4021 in authentication"

Keyword (BM25): finds docs containing "E-4021" → high score
Vector: finds docs about "authentication errors" → high score

Hybrid: the document about "E-4021 in the auth module" scores highest on BOTH
  → Ranked #1 with high confidence
```

## BM25: The Keyword Side

### ELI5

BM25 is a smarter version of "count how many times the search word appears." It considers:
1. **How often the word appears in THIS document** (more = more relevant)
2. **How rare the word is across ALL documents** (rare words matter more)
3. **How long the document is** (longer docs get more matches by chance)

### The Formula (Simplified)

```
BM25(query, document) = Σ  IDF(term) × TF(term, doc) × (k1 + 1)
                                        ─────────────────────────
                                        TF(term, doc) + k1 × (1 - b + b × |doc|/avgdl)

Where:
  IDF = Inverse Document Frequency (how rare is this word?)
  TF  = Term Frequency (how often does it appear in this doc?)
  k1  = Term frequency saturation (default 1.2)
  b   = Length normalization (default 0.75)
  |doc| = document length
  avgdl = average document length
```

**Key intuition:**
- "the" appears in every document → low IDF → low score
- "E-4021" appears in 1 document → high IDF → high score
- A word appearing 10 times vs 100 times → diminishing returns (saturation)

## Sparse Vectors: How Qdrant Implements Keyword Search

Qdrant supports hybrid search through **sparse vectors** — a representation of BM25/keyword relevance.

### Dense vs Sparse Vectors

```
Dense vector (embedding):
  [0.12, -0.45, 0.78, 0.33, -0.21, 0.56, ...]  (ALL dimensions have values, 768 dims)

Sparse vector (keyword):
  {42: 1.5, 1337: 0.8, 9001: 2.1}  (only non-zero dimensions, mapped to vocabulary IDs)

  Where:
    42   → vocabulary index for "python"  → weight 1.5
    1337 → vocabulary index for "machine" → weight 0.8
    9001 → vocabulary index for "learning"→ weight 2.1
```

Sparse vectors are generated by models like **SPLADE** or **BM25** encoders, or by libraries like **fastembed**.

### Setting Up Hybrid Search in Qdrant

```python
from qdrant_client import QdrantClient
from qdrant_client.models import (
    VectorParams, SparseVectorParams, Distance,
    PointStruct, SparseVector, NamedVector, NamedSparseVector,
    Prefetch, FusionQuery, Fusion,
)

client = QdrantClient(host="localhost", port=6333)

# Create collection with BOTH dense and sparse vectors
client.create_collection(
    collection_name="articles_hybrid",
    vectors_config={
        "dense": VectorParams(size=768, distance=Distance.COSINE),
    },
    sparse_vectors_config={
        "sparse": SparseVectorParams(),
    },
)
```

### Generating Sparse Vectors

**Option 1: Using fastembed (recommended)**

```bash
uv add fastembed
```

```python
from fastembed import SparseTextEmbedding, TextEmbedding

# Dense model
dense_model = TextEmbedding("BAAI/bge-small-en-v1.5")

# Sparse model (SPLADE)
sparse_model = SparseTextEmbedding("prithivida/Splade_PP_en_v1")

text = "Python is a programming language for machine learning"

# Generate dense embedding
dense_emb = list(dense_model.embed([text]))[0]  # [0.12, -0.45, ...]

# Generate sparse embedding
sparse_emb = list(sparse_model.embed([text]))[0]
# SparseEmbedding(indices=[42, 1337, 9001, ...], values=[1.5, 0.8, 2.1, ...])
```

**Option 2: Using Qdrant's built-in BM25**

Qdrant can compute sparse vectors from text automatically using payload fields indexed with `TEXT` type:

```python
# Index a text field for full-text search
client.create_payload_index(
    collection_name="articles_hybrid",
    field_name="text",
    field_schema="text",  # enables BM25-style tokenization
)
```

### Inserting Hybrid Points

```python
# With fastembed-generated sparse vectors
client.upsert(
    collection_name="articles_hybrid",
    points=[
        PointStruct(
            id=1,
            vector={
                "dense": dense_emb.tolist(),
            },
            payload={
                "title": "Intro to ML",
                "text": "Machine learning is a subset of AI...",
            },
        ),
    ],
)

# Update sparse vector separately
client.update_vectors(
    collection_name="articles_hybrid",
    points=[
        {
            "id": 1,
            "vector": {
                "sparse": SparseVector(
                    indices=sparse_emb.indices.tolist(),
                    values=sparse_emb.values.tolist(),
                ),
            },
        },
    ],
)
```

Or insert both at once:

```python
client.upsert(
    collection_name="articles_hybrid",
    points=[
        PointStruct(
            id=1,
            vector={
                "dense": dense_emb.tolist(),
                "sparse": SparseVector(
                    indices=sparse_emb.indices.tolist(),
                    values=sparse_emb.values.tolist(),
                ),
            },
            payload={"title": "Intro to ML", "text": "Machine learning is..."},
        ),
    ],
)
```

## Fusion: Combining Dense + Sparse Results

The key question: you have two ranked lists (dense and sparse). How do you merge them into one?

### Reciprocal Rank Fusion (RRF)

```
For each document appearing in any list:
  RRF_score = Σ  1 / (k + rank_in_list_i)

  Where k = 60 (constant, prevents division by very small numbers)
```

**Example:**

```
Dense results:  [Doc_A (rank 1), Doc_B (rank 2), Doc_C (rank 3)]
Sparse results: [Doc_B (rank 1), Doc_D (rank 2), Doc_A (rank 3)]

RRF scores (k=60):
  Doc_A: 1/(60+1) + 1/(60+3) = 0.0164 + 0.0159 = 0.0323
  Doc_B: 1/(60+2) + 1/(60+1) = 0.0161 + 0.0164 = 0.0325  ← highest!
  Doc_C: 1/(60+3) + 0         = 0.0159
  Doc_D: 0         + 1/(60+2) = 0.0161

Final ranking: [Doc_B, Doc_A, Doc_D, Doc_C]
```

Doc_B wins because it appears highly in BOTH lists.

### Hybrid Search in Qdrant (Prefetch + Fusion)

```python
# Hybrid search with RRF fusion
results = client.query_points(
    collection_name="articles_hybrid",
    prefetch=[
        # Dense search (semantic)
        Prefetch(
            query=dense_query_embedding,
            using="dense",
            limit=20,
        ),
        # Sparse search (keyword)
        Prefetch(
            query=SparseVector(
                indices=sparse_query.indices.tolist(),
                values=sparse_query.values.tolist(),
            ),
            using="sparse",
            limit=20,
        ),
    ],
    query=FusionQuery(fusion=Fusion.RRF),  # combine with RRF
    limit=10,
)
```

### Distribution-Based Score Fusion (DBSF)

An alternative to RRF that normalizes scores based on their distribution:

```python
results = client.query_points(
    collection_name="articles_hybrid",
    prefetch=[
        Prefetch(query=dense_query, using="dense", limit=20),
        Prefetch(query=sparse_query, using="sparse", limit=20),
    ],
    query=FusionQuery(fusion=Fusion.DBSF),  # distribution-based
    limit=10,
)
```

**RRF vs DBSF:**

| Aspect | RRF | DBSF |
|--------|-----|------|
| Uses scores | No (rank only) | Yes (normalizes scores) |
| Sensitivity | Stable, predictable | More sensitive to score distributions |
| When to use | Default choice | When score magnitude matters |

## Weighted Hybrid Search

Control the balance between dense and sparse:

```python
# Heavier on semantic (dense)
results = client.query_points(
    collection_name="articles_hybrid",
    prefetch=[
        Prefetch(query=dense_query, using="dense", limit=20, score_threshold=0.5),
        Prefetch(query=sparse_query, using="sparse", limit=20),
    ],
    query=FusionQuery(fusion=Fusion.RRF),
    limit=10,
)
```

For more control, use **re-scoring** with a custom formula or re-ranker (covered in Phase 3 — RAG).

## Full-Text Search (Without Sparse Vectors)

For simpler hybrid search, Qdrant supports full-text search on TEXT-indexed payload fields:

```python
from qdrant_client.models import FieldCondition, MatchText

# Create text index on a field
client.create_payload_index(
    collection_name="articles",
    field_name="text",
    field_schema="text",
)

# Search: vector similarity + text match filter
results = client.query_points(
    collection_name="articles",
    query=[0.1, 0.2, ...],
    query_filter=Filter(
        must=[
            FieldCondition(
                key="text",
                match=MatchText(text="error E-4021"),
            ),
        ]
    ),
    limit=10,
)
```

This is simpler than sparse vectors but less powerful — it's a hard filter (include/exclude) rather than a scored ranking.

## When to Use What

```
Simple use case (< 1M docs, no rare terms):
  → Dense vectors + payload filters
  → Good enough for most applications

Production with technical/domain content:
  → Hybrid: Dense + Sparse (SPLADE)
  → RRF fusion
  → MANDATORY for search quality

Maximum quality:
  → Hybrid search
  → + Re-ranking with cross-encoder (Phase 3)
```

---

## Summary

| Concept | Key Point |
|---------|-----------|
| Vector search | Finds semantic matches, misses exact terms |
| Keyword search (BM25) | Finds exact matches, misses synonyms |
| Hybrid search | Combines both — mandatory for production quality |
| Sparse vectors | SPLADE/BM25 representations for keyword relevance |
| RRF | Merges ranked lists using reciprocal ranks (default choice) |
| DBSF | Score-aware fusion (alternative to RRF) |
| Prefetch | Qdrant's mechanism to run multiple searches, then fuse |
| Full-text filter | Simpler alternative — hard filter, not scored ranking |
